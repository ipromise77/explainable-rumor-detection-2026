# 大模型 RAG 与 Prompt 迭代实验记录

引入交大提供的 deepseek-reasoner 接口后，我们进行了多轮 Prompt 和策略调整。大模型接入后并非直接带来准确率提升，初期甚至出现了反向效果。

## 1. Prompt 演进

本地基座模型（TF-IDF + LR）准确率为 88.03%，混淆矩阵显示有 37 条谣言被漏报（FN=37）。为了处理这些样本，我们对基座模型预测概率在 [0.30, 0.70] 之间的摇摆样本，调用 DeepSeek 进行 RAG（检索增强生成）复核。

第一版 Prompt 让大模型直接结合相似推文进行判断。结果大模型对包含敏感词的突发新闻普遍判为谣言，准确率从 88.03% 下降到 87.28%。

## 2. 引入对比式 RAG 检索

最初只检索 4 条最相似的推文给大模型参考，但发现如果检索结果中真假样本混合，大模型的判断会出现偏差。后来改为对比式 RAG：同时展示最相似的真新闻和最相似的谣言，让大模型有更完整的参考信息。

## 3. 非对称置信度门槛

为了防止大模型在不准确的情况下修改本地模型标签，我们设置了非对称覆盖阈值：将非谣言翻转为谣言需要置信度 >= 0.75，将谣言翻转为非谣言需要置信度 >= 0.70。调用区间优化为 [0.20, 0.80]。

## 4. 结果

在验证集上，大模型共被调用 70 次。在严格的覆盖阈值下，最终只触发了 3 次标签翻转。准确率从 88.03% 提升到 88.28%，仅救回了 2 个漏报样本。仍有 35 个 FN 未被处理。

## 5. 可能的原因

(1) 大模型在缺乏背景信息时容易产生幻觉。为了控制误判，我们设置了较高的覆盖阈值。结果在面对 35 个难以分辨的样本时，大模型倾向于给出低置信度，不触发覆盖。

(2) 推文数据本身短且碎片化。这些基座模型无法正确分类的样本，在纯文本信息层面可能已经缺乏足够的区分信号。在不具备实时联网查证能力的条件下，仅靠文本特征难以判断真伪。

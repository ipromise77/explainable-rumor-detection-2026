# 大模型 RAG 与 Prompt 迭代实验记录

在引入学校提供的 `deepseek-reasoner` 接口后，我们做了多轮复核与解释生成实验。核心结论是：大模型并不是接入后就一定提分，必须用保守门槛、相似样本证据和本地模型概率共同约束。

## 1. Prompt 演进

本地基座模型（TF-IDF + Logistic Regression 集成）在 `val.csv` 上 Accuracy 为 0.8803。混淆矩阵显示存在 37 条 false negative。为了改善解释质量并尝试救回漏报，我们引入 DeepSeek 接口，对本地模型低置信样本进行 RAG 复核。

早期版本曾让大模型更自由地改标签，结果大模型容易把包含敏感事件词的真实新闻误判为谣言，准确率反而下降。因此后续策略改为：本地模型负责主要分类，大模型主要负责解释；只有在高置信冲突时才允许覆盖。

## 2. RAG 相似样本提示

LLM 提示中包含本地模型概率、支持谣言和支持非谣言的局部证据，以及若干相似训练样本。这样做的目的不是让大模型凭空判断事实，而是让解释更贴近本数据集的文本分布。

后续还尝试过“对比式 RAG”，即同时给出相似非谣言和相似谣言样本。但该方向尚未进入最终主线代码，当前正式代码仍采用轻量的相似样本检索方案。

## 3. 覆盖门槛

最终主线保留统一的高置信覆盖门槛：当大模型与本地模型冲突，且大模型自评置信度不低于 0.85 时，才允许覆盖本地标签。该策略牺牲了一部分召回，但能显著降低大模型误改标签的风险。

另有低置信 false negative 召回复核实验：只复核本地判 0 且 `0.20 <= prob_rumor < 0.50` 的样本，仍使用 0.85 高置信覆盖门槛。该实验救回 1 条 false negative，没有新增 false positive。

## 4. 结果

通用 LLM 增强策略在验证集上调用 70 条低置信样本，最终覆盖 3 条，Accuracy 从 0.8803 小幅提升到 0.8828，Macro-F1 从 0.8757 提升到约 0.8787。

低置信 false negative 召回复核策略调用 79 条候选样本，最终覆盖 1 条，Accuracy 同样达到 0.8828，Rumor F1 从 0.8519 提升到 0.8554。

## 5. 反思

1. 大模型在缺乏外部事实核查时，不能可靠判断推文内容真假，因此不能让它无约束地接管标签。
2. 短推文本身信息量有限，很多 false negative 仅凭文本字面很难判断。
3. LLM 更适合在本项目中承担“解释增强”和“高置信少量复核”，而不是大规模替代本地分类器。

# 大模型 RAG 与 Prompt 迭代实验记录

在引入交大提供的 deepseek-reasoner 接口后，我们经历了一个非常痛苦但收获巨大的调优过程。大模型并不是接入了就能直接提分的，甚至一度导致了反向优化。

## 1. Prompt 演进

我们的本地基座模型（TF-IDF + LR）准确率卡在 88.03%，通过混淆矩阵我们发现有 37 条谣言被漏报（假阴性）。为了把这些“伪装极好”的谣言揪出来，我们决定引入 DeepSeek 接口，对基座模型预测概率在 [0.30, 0.70] 之间的摇摆样本进行 RAG（检索增强生成）复核。
第一次优化决定直接让大模型结合相似推文进行判断。结果大模型表现得极度多疑，把许多包含敏感词的真实突发新闻全判成了谣言。这导致我们的准确率不升反降，直接跌到了 87.28%。

## 2. 引入对比式 RAG 检索

原本我们只给大模型检索 4 条最相似的推文，但发现如果这 4 条里真假参半，大模型就会失去作用。后来我们改进了检索逻辑，升级为对比式 RAG，强制同时给它看最相似的真新闻和最相似的谣言。

## 3. 确立非对称置信度门槛

为了防止大模型过度自信地乱改标签，我们在代码中定下了规矩：大模型如果想把非谣言推翻成谣言，必须给出 0.75 及以上的绝对置信度；但如果它觉得某个推文是被冤枉的，想要洗白成非谣言，只需要 0.70 的置信度即可。结合最终优化到 `[0.20, 0.80]` 的调用区间


## 4. 结果
在验证集上，大模型总共被触发调用了 70 次，但在严苛的门槛下，它最终只敢出手推翻 3 次本地判断。
最终准确率仅仅从 88.03% 极其微弱地爬到了 88.28%，仅仅捞回了2个样本。还有足足 35 个假阴性样本。

## 5. 可能的原因
（1）大模型本身在缺乏足够背景信息时，很容易产生幻觉和误判。我们为了把跌下去的准确率拉回来，给它加了极其苛刻的保守限制。结果面对那 35 个真正难以分辨的困难样本，它宁愿输出低置信度选择服从基座，也不敢承担推翻错误的风险。

（2）推特数据本身极短且碎片化。这 35 个基座模型分不出来的困难样本，可能在字面信息上已经达到了“信息熵的极限”。单纯依靠文本字面特征，哪怕是大模型，在不具备实时联网查证能力的情况下，也无法判断一句话到底是真是假。

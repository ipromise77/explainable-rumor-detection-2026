# 实验与调优记录

## 1. 动态阈值扫描

一开始，使用了默认的 0.5 作为逻辑回归的判定界限。但在分析混淆矩阵时，我发现基座模型有点偏保守，漏报了不少真实的谣言样本。
为了解决这个问题，我写了一个暴力的循环代码，动态扫描最佳的判定阈值。经过测试，我们发现将阈值设定为 0.54 时，模型对漏报的校准效果最好。但是基座模型的准确率提高极其有限，完全不显著。

## 2. 强融 Event 特征,失败

考虑到谣言通常具有极强的事件聚集性（比如相同的突发事件会有相似的造谣话术），我一开始兴致勃勃地想要把数据集里的 `event` 字段利用起来。
尝试对 `event` 进行了 One-Hot 编码，并硬拼接到了基于文本提取的高维 TF-IDF 稀疏矩阵中 。本以为这能大幅提分，结果验证集准确率反而从 0.8803 微降到了 0.8753 
经过复盘和查阅资料，总结如下。因为我们的词级与字符级 TF-IDF 提取了数万维的特征，而事件特征只有寥寥几维。在训练时，这几个可怜的维度完全被文本特征淹没，不仅没发挥作用，反而导致了模型对某些训练样本的轻微过拟合 。虽然这条路没走通，但让我们对经典线性模型的局限性有了更直观的认识。

## 3. 尝试 L1 正则化降噪失败

在特征工程阶段，由于我们提取了近 8 万维的特征，我们怀疑其中包含大量无意义的噪声。为了过滤这些杂音，我们计划将逻辑回归默认的 L2 正则化替换为 L1 稀疏正则化（Lasso），希望强行将无用特征的权重压缩为 0。
然而在实际实验过程中，我们发现 L1 正则化需要使用特定求解器（如 liblinear 或 saga），而在近 8 万维的高维稀疏特征下，训练时间明显增长，并出现收敛困难的问题。考虑到课程项目对复现稳定性的要求，我们最终保留了默认的 L2 正则化方案。

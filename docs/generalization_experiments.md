# 泛化能力与阈值实验记录

## 1. 实验目的

老师要求模型具有一定泛化能力。为了避免只报告 `val.csv` 单一准确率，本项目新增三类补充实验：

1. 事件级准确率分析：观察模型在不同 `event` 主题上的稳定性。
2. 训练集内部 dev 阈值实验：只使用 `train_clean.csv` 内部切分选择阈值，避免直接在 `val.csv` 上调参。
3. 5-fold 交叉验证：在清洗训练集上评估模型表现的波动范围。

所有实验由 `scripts/run_generalization_experiments.py` 生成。

## 2. 事件级准确率分析

使用最终本地模型在 `val.csv` 上按 `event` 分组统计。结果保存到 `results/event_accuracy.csv`，其中 event 0 / event 1 的逐条错误分析额外保存到 `results/event_0_1_error_analysis.csv`。

| event | support | non-rumor | rumor | Accuracy | Macro-F1 | Rumor F1 |
| ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 0 | 13 | 7 | 6 | 0.7692 | 0.7451 | 0.6667 |
| 1 | 109 | 85 | 24 | 0.8532 | 0.7070 | 0.5000 |
| 2 | 1 | 0 | 1 | 1.0000 | 0.5000 | 1.0000 |
| 3 | 22 | 0 | 22 | 1.0000 | 0.5000 | 1.0000 |
| 4 | 46 | 23 | 23 | 0.8913 | 0.8913 | 0.8889 |
| 5 | 121 | 69 | 52 | 0.8678 | 0.8651 | 0.8462 |
| 6 | 89 | 42 | 47 | 0.9101 | 0.9101 | 0.9091 |

结论：

- 模型在 event 4、5、6 上表现较稳，尤其 event 6 的 Accuracy 和 Macro-F1 均约 0.91。
- event 0 样本数只有 13，准确率最低，为 0.7692，说明少样本事件更容易波动。
- event 1 的整体 Accuracy 为 0.8532，但 Rumor F1 只有 0.5000，主要原因是该事件中非谣言样本较多，模型对谣言召回不足。
- event 2 和 event 3 的 Accuracy 为 1.0，但 event 2 只有 1 条、event 3 全部为谣言，因此不能过度解读为泛化完美。

### 2.1 event 0 / event 1 错误分析

由于 event 0 准确率最低、event 1 的谣言类 F1 最低，我们进一步导出这两个事件的逐条预测结果、样本序号、概率、错误类型和局部证据。表格保存到 `results/event_0_1_error_analysis.csv`。

| event | support | 错误数 | False Negative | False Positive | 真实谣言平均谣言概率 | 真实非谣言平均谣言概率 |
| ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 0 | 13 | 3 | 3 | 0 | 0.4542 | 0.1314 |
| 1 | 109 | 16 | 16 | 0 | 0.3890 | 0.0928 |

关键发现：

1. event 0 和 event 1 的错误全部是 `false_negative`，即真实标签为谣言，但模型预测为非谣言。
2. 两个事件没有出现 false positive，说明模型在这些事件上偏保守，宁愿少报谣言，也很少把非谣言误报为谣言。
3. event 1 的真实谣言平均谣言概率只有 0.3890，明显低于 0.5 阈值，解释了该事件 Rumor F1 只有 0.5000 的原因。
4. 后续若继续优化，应优先提升 event 0 / event 1 的谣言召回，例如加入事件级 dev 分析、对低置信 false negative 使用大模型复核，或在不泄漏验证集标签的前提下增加更稳定的语义特征。

## 3. train 内部 dev 阈值实验

为了避免在 `val.csv` 上调阈值造成“测试集调参”，我们从 `train_clean.csv` 内部按标签分层切分：

- inner train：2266 条
- dev：567 条

在 inner train 上训练模型，在 dev 上搜索阈值 `0.20` 到 `0.80`。结果保存到 `results/dev_threshold_experiment.csv`。

dev 上最佳阈值为 `0.51`：

| 阈值 | Dev Accuracy | Dev Macro-F1 | Dev Rumor F1 |
| ---: | ---: | ---: | ---: |
| 0.51 | 0.8536 | 0.8508 | 0.8303 |

但是把该阈值应用到最终模型的 `val.csv` 概率上，表现反而下降：

| 阈值 | Val Accuracy | Val Macro-F1 | Val Rumor F1 |
| ---: | ---: | ---: | ---: |
| 0.50 | 0.8803 | 0.8757 | 0.8519 |
| 0.51 | 0.8753 | 0.8703 | 0.8447 |

结论：虽然 dev 最佳阈值略高于 0.5，但泛化到 `val.csv` 后不如默认阈值。因此最终保留 `0.5`，避免为了小幅 dev 波动牺牲测试集稳定性。

## 4. 5-fold 交叉验证

在 `train_clean.csv` 上做 5-fold StratifiedKFold，结果保存到 `results/cv_results.csv`。

| Fold | Accuracy | Macro-F1 | Rumor F1 |
| ---: | ---: | ---: | ---: |
| 1 | 0.8360 | 0.8311 | 0.8025 |
| 2 | 0.8642 | 0.8608 | 0.8392 |
| 3 | 0.8801 | 0.8780 | 0.8623 |
| 4 | 0.8622 | 0.8578 | 0.8326 |
| 5 | 0.8781 | 0.8751 | 0.8559 |

汇总：

| 指标 | Mean | Std | Min | Max |
| --- | ---: | ---: | ---: | ---: |
| Accuracy | 0.8641 | 0.0176 | 0.8360 | 0.8801 |
| Macro-F1 | 0.8606 | 0.0187 | 0.8311 | 0.8780 |
| Rumor F1 | 0.8385 | 0.0235 | 0.8025 | 0.8623 |

结论：5-fold 平均 Accuracy 为 0.8641，标准差约 0.0176，说明模型在不同训练子集上表现较稳定；最终 `val.csv` Accuracy 为 0.8803，处于交叉验证波动范围上沿，但没有明显脱离训练集内部验证表现。

## 5. 总体结论

本轮实验支持以下决策：

1. 保持默认分类阈值 `0.5`，因为 train 内部 dev 最佳阈值 `0.51` 在 `val.csv` 上变差。
2. 报告中除 `val.csv` Accuracy 外，补充 5-fold 交叉验证均值和事件级表现，增强“泛化能力”论证。
3. 后续如果继续提升，可重点关注 event 0 和 event 1 的错误样本，因为它们暴露出少样本事件波动和谣言召回不足问题。
